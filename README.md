# GourmetOS

**The ultimate operating system for modern restaurants.**

GourmetOS is a comprehensive, all-in-one restaurant management platform designed to streamline operations, enhance guest experiences, and provide real-time insights for owners.

## 🚀 Key Features

*   **Owner Dashboard:** Real-time metrics including revenue, labor cost, table turnover, and top-selling items.
*   **Visual Table Management (Canvas):** Interactive drag-and-drop floor plan editor.
*   **Smart Host Station:** Manage reservations, waitlists, and seat guests with a live floor view.
*   **Waiter POS:** Fast, touch-friendly Point of Sale interface for taking orders tableside.
*   **Menu Management:** Easy-to-use tools for updating dishes, prices, and availability found in `lib/demo-data.ts`.
*   **Digital Staff Management:** Track staff roles, status, and contact information.
*   **Premium Booking Experience:** Customer-facing reservation page with immersive design.

## 🛠️ Tech Stack

*   **Framework:** [Next.js 16](https://nextjs.org/) (App Router)
*   **Language:** TypeScript
*   **Styling:** Tailwind CSS + Lucide Icons + Shadcn UI (Components)
*   **Charts:** Recharts
*   **State/Drag & Drop:** `@dnd-kit`
*   **Icons:** Lucide React

## 📦 Getting Started

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/your-username/gourmet-os.git
    cd gourmet-os
    ```

2.  **Install dependencies:**
    ```bash
    npm install
    ```

3.  **Run the development server:**
    ```bash
    npm run dev
    ```

4.  **Open your browser:**
    Navigate to [http://localhost:3000/dashboard](http://localhost:3000/dashboard) to see the app in action.

## 🚀 Deployment

This project is optimized for deployment on **Vercel**.

1.  Push your code to a GitHub repository.
2.  Import the project into Vercel.
3.  Vercel will detect Next.js and configure the build settings automatically.
4.  Deploy!

## 📂 Project Structure

*   `/app`: Next.js App Router pages and layouts.
*   `/components`: Reusable UI components.
*   `/lib/demo-data.ts`: Mock data for the application (connected to all views).
*   `/types`: TypeScript type definitions.
